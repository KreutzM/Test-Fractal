# Connector workflow

This note describes how the repository can be evaluated through the GitHub connector.

The connector reads issues, pull requests, repository files, and diffs.

The local machine runs the orchestrator, Codex, Git, and local tests.

The person using the system makes final project decisions.

For an issue, first check goal, scope, non-goals, acceptance criteria, and test notes.

For a pull request, read metadata, changed files, diff, agent report, and test notes.

Useful entry files are README.md, AGENTS.md, config/orchestrator.json, docs/PLAN.md, and docs/ARCHITECTURE.md.
